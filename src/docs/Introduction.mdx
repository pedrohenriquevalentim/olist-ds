# Olist Design System

Sistema de componentes React + TypeScript construído sobre design tokens exportados do Figma via Style Dictionary.

## Instalação

```bash
npm install @pedrohenriquevalentim/olist-ds
```

## Uso básico

Importe os tokens CSS globalmente **uma vez** na raiz da aplicação:

```tsx
import '@pedrohenriquevalentim/olist-ds/dist/variables.css';
```

Em seguida, importe os componentes onde precisar:

```tsx
import { Button, Chip, Icon } from '@pedrohenriquevalentim/olist-ds';
```

## Componentes disponíveis

| Componente | Categoria | Descrição |
|---|---|---|
| `Button` | Action | Botão com variantes primary, secondary e tertiary |
| `Checkbox` | Input | Caixa de seleção com rótulo e estado de erro |
| `Chip` | Input / Filter | Tag interativa para filtros e seleção múltipla |
| `Icon` | Brand | Ícones do design system (rebrand 24px) |
| `InputSelect` | Input | Campo de seleção com suporte a busca e múltipla seleção |
| `Logo` | Brand | Logotipo Olist em variantes de cor e tamanho |
| `ProdutosOlistIcons` | Brand | Ícones dos 8 produtos Olist com estados e temas |

## Foundations

| Seção | O que documenta |
|---|---|
| **Colors** | Paleta completa com tokens primitivos e semânticos |
| **Typography** | Escala de tamanhos, pesos, alturas de linha e papéis de texto |
| **Spacing** | Grid de 4px, espaçamentos, border-radius e espessuras de borda |

## Princípios

- **Consistência** — Todos os componentes consomem exclusivamente design tokens do Figma
- **Acessibilidade** — WCAG 2.1 AA em todos os componentes interativos
- **Composição** — Componentes pequenos que se combinam em telas complexas
- **Clareza** — Interface orientada a ferramenta; sellers usam 8h/dia, clareza > decoração
